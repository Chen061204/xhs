# Gemini 前端实现规范

请在现有 Next.js 单页应用和新粗野主义视觉体系内实现以下交互。不要创建仅用于展示、
点击无响应的按钮，也不要修改后端 API JSON 合约。

## 1. 左侧创作工作流

左侧保留五个步骤，并把它们实现为同一份分析结果的分段导航：

1. 今日爆款雷达：切换到热点列表并滚动到页面顶部。
2. 爆款深度拆解：切换到分析视图并滚动到 `AI 深度诊断`。
3. 生成衍生选题：滚动到三条差异化创作方向与方向切换按钮。
4. 一键生成文案：滚动到当前方向的小红书正文模板。
5. 多模态提示词：滚动到当前方向的 Gemini 图片提示词和 Gemini Veo 视频提示词。

交互要求：

- 当前步骤必须有黄色背景、黑色投影和 `aria-current="step"` 激活态。
- 尚未执行“拆解并生成”时点击第 2-5 步，不得无响应；显示
  “请先从热点列表选择选题并生成拆解”的错误提示，并返回热点雷达。
- 分段跳转使用平滑滚动，并为吸顶顶部栏保留 scroll margin。
- 桌面端为纵向步骤条；窄屏保持横向可滚动步骤条。

## 2. 顶部赛道分类

顶部保留“今日总榜、美妆穿搭、生活方式、职场成长、旅行美食”五个分类。

- 点击没有本地缓存的分类时，立即调用
  `GET /api/trending?limit=10&category=<分类>`；“今日总榜”不传 category。
- 请求期间禁用所有分类按钮，在当前分类按钮中显示旋转加载图标。
- 点击已有缓存的分类时直接恢复结果，不重复消耗联网搜索额度。
- 当前分类使用黄色背景、黑色投影和 `aria-pressed="true"`。
- 用户点击“扫描联网热点”时，强制刷新当前分类并覆盖该分类缓存。

## 3. 热点结果持久化

不要提供任何 demo 热点或 example.com 演示来源。

- 使用版本化的 `localStorage` key 保存每个分类的 `items`、`date`、
  `disclaimer` 和 `savedAt`。
- 刷新页面时恢复上次查看的分类和扫描结果。
- 缓存损坏时安全删除缓存并显示空状态，不能让页面崩溃。
- 空状态应明确提示用户首次扫描；扫描成功后显示 `SAVED` 和保存时间。
- API Key 继续只保存在当前浏览器，不发送到任何非后端地址。

## 4. Gemini 多模态结果 UI

- 标题明确写为 `GEMINI IMAGE PROMPT` 与 `GEMINI VEO PROMPT`。
- 图片、视频提示词均提供独立复制按钮。
- 三个衍生方向切换后，正文、图片提示词和视频提示词必须同步切换。
- 不在前端拼接或伪造提示词；直接展示后端 `/api/analyze` 的结构化结果。

## 5. 验收标准

- 五个左侧步骤均有可观察到的行为。
- 五个分类均能切换；无缓存时联网，有缓存时秒开。
- 页面刷新后热点结果仍存在。
- 首次打开且无缓存时列表为空，不出现 demo 数据。
- `pnpm build` 与 TypeScript 检查通过，桌面和移动端布局均无溢出。
